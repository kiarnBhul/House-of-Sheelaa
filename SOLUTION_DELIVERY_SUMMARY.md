## üéì Key Concepts Explained

### 3-Layer Persistence Architecture
```
Layer 1: RAM (In-Memory)
‚îú‚îÄ Speed: INSTANT (<100ms)
‚îú‚îÄ Persistence: Lost on restart ‚ùå
‚îî‚îÄ Use: Active app operations

Layer 2: Phone Storage (SharedPreferences)
‚îú‚îÄ Speed: FAST (<500ms)
‚îú‚îÄ Persistence: Survives restart ‚úÖ
‚îî‚îÄ Use: Local caching

Layer 3: Cloud (Firestore)
‚îú‚îÄ Speed: SLOWER (<2s)
‚îú‚îÄ Persistence: Survives reinstall ‚úÖ
‚îî‚îÄ Use: Backup & cross-device sync
```

### Loading Priority on App Start
1. Try RAM (app just started, empty)
2. Try Local Storage (SharedPrefs) ‚Üí INSTANT LOAD ‚úÖ
3. Try Firestore backup (if local missing)
4. Fetch from Odoo (if all else fails)

### Automatic Saving Pattern
```
User Action
  ‚Üì
Save to RAM (immediate)
  ‚Üì

  ‚Üì
Background: Upload to Firestore (async)
```
